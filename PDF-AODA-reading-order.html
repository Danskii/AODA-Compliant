Part 3: Reading Order

Adobe Acrobat and accessibility devices use PDF tags to determine in what order the content of the page should be read.

Unlike a sighted human reader who can usually infer the flow of content from one paragraph to another an accessibility devices requires explicit instructions. These instructions come from the PDF's tags.

Additionally tags help identify what type of content is contained within a PDF: paragraphs, headings, images and more. All are identified with tags.

Here's how you can build a tag structure.

  1. Click on the tags option in the left hand navigation pane
  2. Click on the drop down menu above the tags tree
  3. Select highlight content

If your document contains no tags please follow the next steps. Otherwise if your document has tags this section will show you how to edit them.

Building Reading Order With Tags

  1. Right click on the "No Tags available" text
  2. Select "Create Tags Root"
  3. Save
  4. Select the "Accessibility" tool from the right hand panel
  5. Select the "Reading Order" tool
  6. Right click on the "Tags" root you created and select "New Tag" 
  7. Select the type of Tag you'd like to create and click Ok. The three most common tags to use are: P, H and Figure. Use "P" to create a paragraph tag. Use this tag to identify paragraphs.
    8. Use "H1, H2, H3, H4, H5, H6" to create a heading tag. Heading tags should be used sequentially with H1 being the highest level heading and H6 the lowest. Use "Figure" to create an image tag. This tag is used to identify images as well as to add alternate text to the image.
  8. Use your mouse to draw a box around the content you'd like to tag
  9. Right click on the tag you just created and select "Create Tag from Selection"
  10. You should now see the tag you just created in the "Tags" panel to the left

Editing Existing Reading Order Tags

  1. Click on the + or > (depending on your version of Acrobat) next to the Tags root, in the left hand tags panel, to expand the tag tree.
  2. NOTE: The order of tags in the tags tree you just expanded is the order the content will be read or interpreted by an accessibility device. In my example screenshot the tags will proceed in this order:
    1. p
    2. p
    3. h2
    4. p
    5. p
    6. h1
  3. To adjust the reading order left click on the tag you would like to shift to another position
  4. Press control+X to cut the tag
  5. Use the up and down arrow keys on your keyboard to move the cursor above the position where you'd like to shift the tag to
  6. Press control+V to paste the tag into its new position
  7. NOTE: You can move through the tags, from top to bottom, while watching the highlight cursor move through the text on the page. This is a good way to verify that the reading order is the way you want it to be.

Ok, that's it for this section.
